The Shift

Most people think of AI tools as products. I used to think that way too.

Claude Code has Claude inside it. Cursor runs on GPT. Copilot is GitHub's thing. The interface and the intelligence feel like one indivisible object — you pick a tool, and you inherit its model, its pricing, its privacy posture, and its politics.

Then something changed. In January 2026, Ollama announced support for the Anthropic Messages API. What that means in plain language: Claude Code — Anthropic's agentic terminal — can now talk to any model running locally on your machine. Same interface. Entirely different brain.

"The agent continues to handle planning, code navigation, and edits. The underlying model no longer needs to live in Anthropic's infrastructure."

That one architectural change quietly turns a product into a platform. And once you see it, you can't unsee it.

Why Bother

This isn't about finding a "better" model. It's about what cloud models can't give you.

The honest answer is that local models are still weaker. They struggle with complex multi-step reasoning, deep debugging, and architecture decisions. You give something up. The question is whether what you gain — privacy, control, zero cost — is worth it for a given task.

That reframe is the whole insight: not which model, but which model for this task.

The Setup

How to wire Claude Code to a local model — every command, every flag.

This works on macOS, Linux, and Windows (WSL or PowerShell). The end state: you open the Claude Code terminal and a model running entirely on your hardware answers you.

01
Install Ollama

Ollama is the runtime that downloads, manages, and serves local models. It exposes them through an API that Claude Code can talk to.

macOS / Linux 
# Install Ollama in one command
curl -fsSL https://ollama.com/install.sh | sh
Windows (PowerShell) 
# Download and run the installer
winget install Ollama.Ollama
# Or visit: https://ollama.com/download

After installation, verify it's running:

Verify 
ollama --version
# Expected: ollama version 0.14.x or higher
# If you get an error, start the server manually:
ollama serve
⚠ You need Ollama v0.14.0 or later. This version added native Anthropic Messages API compatibility — what makes the Claude Code connection possible.
02
Pull a model that supports tool calling

Claude Code isn't just a chatbot — it reads files, runs bash commands, and edits code. For that agentic loop to work, the local model must support tool calling. Not all open models do.

The two most reliable options right now:

Pull models 
# GLM-4.7-Flash — recommended for most setups
# 30B MoE, only 3B active params, 128K context, native tool calling
ollama pull glm-4.7-flash

# Qwen3 Coder — strong alternative, especially for code tasks
ollama pull qwen3-coder

# Verify model is available
ollama list

GLM-4.7-Flash is the current community favourite for this setup. Its Mixture-of-Experts architecture means it punches above its weight on RAM — activating only 3 billion parameters at inference time despite being a 30B model.

03
Install Claude Code

If you don't already have Claude Code, install it with:

macOS / Linux / WSL 
curl -fsSL https://claude.ai/install.sh | bash
Windows CMD 
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
04
Launch Claude Code via Ollama (the easy path)

Ollama's built-in launch command handles all the environment configuration automatically. This is the recommended approach.

Launch — any platform 
# Launch with model selection menu (arrow keys to choose)
ollama launch claude

# Or specify the model directly and skip the menu
ollama launch claude --model glm-4.7-flash

# Auto-pull the model if not already downloaded
ollama launch claude --model glm-4.7-flash --yes

When Claude Code opens, verify it's using your local model:

Inside Claude Code 
# Type this inside the Claude Code terminal
/model
# Output should show: glm-4.7-flash (or your chosen model)
05
The manual path — set environment variables directly

If you prefer to configure things yourself, or if the ollama launch command isn't available in your version, you can set three environment variables that redirect Claude Code's traffic to localhost.

macOS / Linux (bash / zsh) 
# These three vars tell Claude Code: don't call Anthropic's API,
# call Ollama on localhost instead.

export ANTHROPIC_AUTH_TOKEN="ollama"
export ANTHROPIC_API_KEY=""
export ANTHROPIC_BASE_URL="http://localhost:11434"

# Now launch Claude Code pointing at your model
claude --model glm-4.7-flash
Windows PowerShell 
$env:ANTHROPIC_AUTH_TOKEN = "ollama"
$env:ANTHROPIC_API_KEY = ""
$env:ANTHROPIC_BASE_URL = "http://localhost:11434"

claude --model glm-4.7-flash

What each variable does:

ANTHROPIC_AUTH_TOKEN — set to the string "ollama" (not a real token). This tells Claude Code to skip Anthropic auth entirely and trust the local server.

ANTHROPIC_API_KEY — left empty. No Anthropic account needed for this flow.

ANTHROPIC_BASE_URL — redirects all API calls to your Ollama server running on port 11434.

💡 To make these permanent, add the three export lines to your ~/.bashrc or ~/.zshrc. Switch back to cloud Anthropic models anytime by unsetting them and running claude normally.
06
Fix permissions if Claude Code can't see your files

First-time users often find that Claude Code can't read or write files. This is a permissions setting, not a model issue.

Inside Claude Code 
# Check current permissions
/permissions

# Add Bash tool access and file read/write to Allow rules
# Navigate the permissions menu and enable Bash, Read, Write
⚠ By default, most tools are denied. Enable Bash and file access from the /permissions menu to get the full agentic experience.
The Mental Model

Which model for which task.

The setup is interesting. The thinking it enables is more interesting. Once you can switch models with a single command, you start making deliberate choices you never made before.

🔒

Working with sensitive or proprietary code? Switch to a local model. Legal records, private customer data, internal architecture — none of it leaves your machine. GDPR Article 35, data residency requirements, and basic professional judgment all point in the same direction: keep it local.

⚡

Rapid prototyping, boilerplate, or repetitive coding? A local model is fast, free, and more than capable. Scaffolding a new module, writing tests for known patterns, generating documentation — local handles this well.

🧠

Complex debugging, architecture decisions, or novel reasoning? Go cloud. The capability gap is real. Local models are improving fast, but for genuinely hard problems, the best cloud models still have a significant edge.

The meta-point is that you are now making a deliberate resource allocation decision — cost, privacy, quality — for each task. That's a more mature relationship with these tools than "just use the default."

What This Means

Claude Code is quietly becoming a model-agnostic shell.

The interface — the agentic loop, the file system access, the terminal UX — is Anthropic's and stays Anthropic's. The model behind it doesn't have to be. Today it's Ollama with GLM or Qwen. There are already routers like claude-code-router that let you dynamically switch between Anthropic, OpenRouter, DeepSeek, Gemini, and Ollama from within a single session.

Tomorrow the landscape will look different again. New models will drop. Some will be better than today's cloud options at specific tasks. And the interface — Claude Code, or whatever succeeds it — will let you point at any of them.

"It's not a break from vendor control, but it does loosen the ties. The interface stays. The intelligence evolves."

This is what platform thinking looks like in AI tooling. The value migrates to the layer that orchestrates, not the layer that computes. If you're building on AI tools or advising teams that are — the model-agnostic interface pattern is worth watching closely.

The interface stays.
The intelligence evolves.

Would you trade model quality for privacy and control? The answer probably depends on the task — and now, for the first time, you can make that call yourself.
